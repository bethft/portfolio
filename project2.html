Project Overview

About Eat Together!

Eat Together! It is a student-to-student start-up aimed to match students, rather than using a swipe method it matches students based on similar interests/hobbies and schedules. The app aims to get students to eat a meal together to create new friendships. Students can select events listed that have been posted by the event holder and can decide whether they would like to join based on the time availability and restaurant location.

Define

What Was My Project?

Our team's project aimed to facilitate bonding in new social situations by developing a "would you rather" game feature. This game provides thought provoking questions with hypothetical scenarios that spark deep discussions that can create a bond with new acquaintances.

(03)

Reaserch

Goal

Make new social interactions fun! The new feature will let students participate in a lively and enjoyable game designed for getting acquainted with each other.

(04)

Reaserch

Problem

In the current version of Eat Together, there's no feature for students to break the ice or engage in fun activities to form meaningful bonds with new peers during events. Presently, the app solely facilitates event creation and sign-ups. With this new feature, students can dine together and also participate in engaging questions!

(05)

Ideate

How Might We

How might we design a captivating “Would you rather” game that not only fosters comfort but also ignites joy during mealtime gatherings with other students?

(06)

Research questions

  • What are the strengths/weaknesses of existing "would you rather" games or similar icebreaker apps?
  • What are the offerings provided by existing “would you rather” games or similar icebreaker apps?
  • What preferences do users have regarding the tone and content of would-you rather questions (i.e. should questions be humorous, thought-provoking, serious, etc.)?
(07)

User Flow

(08)

Prototype

Mid-Fidelity

Prior to starting the game, users will receive a brief written guideline to familiarize new users with the new feature. We've incorporated a timer to create a sense of urgency in selecting answers, enhancing excitement. If users require more time, the event holder can extend the timer by 30 seconds. Additionally, we've included a feature that displays the number of users who have responded to each question. Once users are ready to begin the game, the screen will present the question along with two options, allowing users to choose the one they resonate with the most. Upon completing their selection, they will be seamlessly directed to a waiting screen, where they can actively monitor the number of participants in the event who have yet to respond.

(09)

Evaluate

User Testing

During user testing, we found that certain features lacked clarity. Users reported that the "add time" feature in the game had no visible countdown, making it difficult for them to gauge the remaining time accurately. Additionally, users found the "add time" prompt unclear, questioning the duration of time added. The initial screen did present the prompt in a clear enough manner. Users often overlooked the feature guidelines, leading to confusion. Users expressed a desire to see the total number of questions in the game and track their progress. They reported confusion and difficulty in finding an exit option when they wanted to stop playing. Additionally, users were unsure if their responses were successfully submitted and wished to have visibility on the remaining time before the next question.

(10)

Prototype

High Fidelity

User testing provided valuable insights that guided revisions to our mid-fidelity wireframes. Based on user feedback, we made the following changes:

  • Provided a more detailed tutorial within the game interface.
  • The timer features a countdown displayed numerically, alongside an additional button indicating the amount of time that can be added.
  • For clarity, labels were added to the timer and response options, helping users understand each feature's functionality.
  • We introduced an "Exit Game" button, allowing users to easily leave the game at any point without hassle.
  • A question tracker has been implemented, enabling users to track their progress.
  • The game page layout was updated with two separate cards
